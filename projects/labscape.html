Context

Labscape is a VR demo game made with Unity engine and the technologies of hand tracking for Meta Quest headsets. Made by the intern team of the CRVM in 3 weeks, this project's purpose was to show off the possibilities of virtual reality, including the feelings of immersion provided by hand tracking, to the participants of a visiting summer camp on XR.
This realization was my first introduction to game production in the professional field and collaborating with people of different specialties (2D & 3D Art, Game Design) also made me discover how to make various VR interactions using hand tracking and poses. This game is in fact one of my favorite projects to date.

Description

Labscape is a short VR escape game using Hand Tracking in which the player is trapped in a futuristic abandoned laboratory in space and must find a way to escape. To do so, the player has to solve various little puzzles and challenges showcasing VR interaction and physics mechanics such as grabbing objects (with life-like movements or at a distance), moving around, activating touch terminals, shooting with a gun, changing gravity, etc…
The game can be finished relatively fast as it is a demo and only has 2 main rooms : the Lab Hub for the interactions showcase and the Labyrinth for movement and others.

Contribution

As the main gameplay programmer I developed most features in this game, the main ones being the garb interaction, the hand poses for each type of object, the locks and the ways to open them, the shooter and an assembly puzzle.

I also coded the audio manager for the background music and the tutorial voice as well as the gravity feature actionable through touch terminals, terminals which I reused for one of the challenges which consisted of activating 3 terminals in a limited time. Said challenge can be adapted to allow timer reset (challenge-global or terminal-local) after an activation.
Among all those features, the most challenging were the shooter and the assembly puzzle, though the lock took me some time due to all the types that exist for this game (singular key, simultaneous keys, key count, challenge related, etc…).

❮ ❯

The shooter was the feature I had the most fun developing. The actual shooting of a projectile wasn’t actually the issue but more how to trigger the action with hand poses since I had never done such a feature before or anything else using hand tracking for that matter. To learn how to do it, I had to look into the code of a water spray feature found in a template feature from the Meta API. I based my code on the “usage” script after which all that was left to do was the hand poses for the idle and trigger.

The actual most time-consuming and difficult feature I had to make was the assembly puzzle where the player must place objects in specific slots to reconstruct an object. This feature would have taken far less time had I had access to snap zones but did not. Indeed, at the time this project was made, the Meta XR API did not allow for a project to also include the OpenXR API which coincidentally has said snap zones I wanted to use.
Because of that, I had to create my own home-made snap zone system and base my puzzle feature on it. I had to do my best to take all interaction cases into account and the multiple ways an object could “attach” to a zone and the related conflicts like which zone to attach the object to in case it’s in contact with more than one for example.
